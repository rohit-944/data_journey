# ⭐ STAR Stories — Interview Answers

## Story 1: EDA Project (Data Analyst angle)

**Situation:** Wanted to analyze real-world data to find actionable business patterns.

**Task:** Analyze 50K+ records and deliver 5 recommendations that a business team could act on.

**Action:** Cleaned data with Pandas (handled 15% missing values, removed duplicates), performed statistical analysis, created 10+ visualizations to identify trends, and wrote a business report.

**Result:** Identified [X key insight] that could save/earn [Y result]. Published on GitHub with full documentation.

---

## Story 2: Data Pipeline Project (Data Engineer angle)

**Situation:** Needed to build an automated pipeline that collects, transforms, and stores data daily.

**Task:** Design and implement an ETL pipeline from API source to queryable warehouse.

**Action:** Built Python extractor for API, stored raw data in S3, transformed with PySpark, orchestrated with Airflow DAGs on a daily schedule, containerized with Docker.

**Result:** Pipeline processes 100K+ records daily with zero manual intervention. Fully reproducible and documented.

---

## Story 3: dbt Pipeline (Analytics Engineer angle)

**Situation:** Raw data in warehouse was messy — analysts couldn't trust or use it directly.

**Task:** Build clean, tested, documented data models that analysts can self-serve from.

**Action:** Built dbt project with staging → intermediate → mart layers. Added data tests (unique, not_null, relationships), documentation, and custom macros for reusability.

**Result:** Created a reliable analytics layer with 100% test coverage. Any analyst can query mart tables confidently.

---

## Story 4: Power BI Dashboard (Data Analyst angle)

**Situation:** Stakeholders needed a way to track [domain] metrics without asking for manual reports.

**Task:** Build an interactive dashboard that updates automatically and allows self-service exploration.

**Action:** Connected Power BI to data source, built star schema model, created 5-page dashboard with DAX measures (YoY growth, running totals), slicers, and drill-through.

**Result:** Dashboard provides real-time insights. Eliminated weekly manual reporting effort.

## Common Questions — Prepared Answers

### "Tell me about yourself" (90 sec)
> "I'm Rohit, a B.Tech CSE graduate specializing in AI. I've built hands-on skills in SQL, Python, Power BI, dbt, and PySpark through 5 real projects — from EDA to automated data pipelines. I can analyze data for insights, model it for warehouses, and build pipelines to move it reliably. I'm looking for a role where I work with data end-to-end."

### "Why should we hire a fresher?"
> "I may not have industry experience, but my 5 projects mirror real work — EDA with business recommendations, interactive dashboards, dbt data models with tests, and an Airflow pipeline on AWS. I write production-ready SQL, I learn fast, and I'll outwork anyone."

### "Why Data Analyst / AE / DE?"
> "I love turning messy data into something useful — whether that's insights for decisions, clean models for teams, or reliable pipelines at scale. These roles let me do exactly that."

### "What's your biggest weakness?"
> "As a fresher, I lack production experience with large-scale systems. But I've compensated by building projects that simulate real scenarios, and I learn extremely fast on the job."
